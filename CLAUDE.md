# Wingman

## What this project does

A personal automation system that filters emails, news, and social media,
delivering only what matters to Slack. Scripts run in the background on the
main PC using pm2. All code lives here on GitHub.

**Core principle:** Important things come to me. I don't go looking for them.

---

## Tech stack

- **Runtime:** Node.js (JavaScript)
- **Process manager:** pm2 — keeps scripts alive 24/7, restarts them on failure,
  and launches them automatically when the PC boots
- **AI:** Claude Code CLI in non-interactive mode (`claude -p "prompt"`) — no API
  key or additional subscription needed, uses the active Pro/Max session
- **Email:** Microsoft Graph API (Hotmail/Outlook) with OAuth 2.0
- **Notifications:** Slack Incoming Webhooks organized by channel
- **Trends:** RSS feeds + Reddit API + additional sources TBD
- **Environment variables:** `.env` file at the root (never commit)

## pm2 — How it works in this project

pm2 is the process manager. Scripts do NOT run as continuous servers — they run as
**cron jobs** managed by pm2:

- `email-agent` → runs every 20 minutes
- `trends-agent` → runs once a day at 7:30am

Managing processes:

```bash
pm2 start ecosystem.config.js     # start all agents
pm2 list                           # check status
pm2 logs                           # live log stream
pm2 logs email-agent               # logs for a specific agent
pm2 restart email-agent            # restart one agent
pm2 stop all                       # stop everything
pm2 save                           # save current process list
pm2 startup                        # configure auto-start on boot
```

## Development workflow

1. Edit code locally with Claude Code
2. Test: `node agents/email/index.js` (direct execution for testing)
3. If everything works: `git add . && git commit -m "..." && git push`
4. Apply changes: `git pull && pm2 restart all`

No auto-deploy — GitHub pushes don't auto-apply. Manual restart required.

## Decision profile (email agent)

The file `config/profile.md` contains the profile in plain language that defines
how Claude classifies emails. It's loaded dynamically on each cycle.

**To adjust agent behavior:** edit `config/profile.md` and run
`pm2 restart email-agent`. No code changes needed.

---

## Persistent state

The email agent saves processed email IDs in a local JSON file to avoid
processing the same email twice. File: `state/email-seen.json`
(auto-generated, not committed).

## Code conventions

- ES Modules (`import/export`) with `"type": "module"` in package.json
- async/await for all async operations (no callbacks)
- Errors always caught with try/catch and logged — one bad email must not
  crash the whole cycle
- Environment variables always from `process.env`, never hardcoded
- Log prefix per agent: `[email-agent]`, `[trends-agent]`
